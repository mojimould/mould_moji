\tPart[lot]{幾何学・運動学的特性の解析計算}{概要}{%
\paragraph*{背景}
ソフトウェアの観点から見た場合、高精度な加工を実現するためにはモールドや工具の\textbf{幾何学的特性}、さらにはマシニング内での\textbf{運動学的特性}を正確に把握することが不可欠であることは言うまでもない。
マシニングセンタの操作・指示はこうした特性の理解が前提である
%% footnote %%%%%%%%%%%%%%%%%%%%%
\footnote{さらに、\index{CAD}CAD, \index{CAM}CAMを利用するというのであれば、これに加えてCADによる\index{3Dモデリング}3Dモデリングの技術、およびそれに伴うCAMの操作技術も習熟しなければならない。}。
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

　しかし現時点（\DMname 設置時点）において、こうした特性はほとんど体系化されておらず、\textbf{明細ごとに対応}をしている状態にある。
またモールドの場合、これらの特性はほとんどが直線または円で記述されるものであり、その体系化は（複雑ではあるが）難しいものではない。

　したがって、（少なくとも実際の加工に関わる程度に）これらの特性を\textbf{把握・体系化}することで、品質や生産性が大きく向上されることが期待される。
\tcbline*
\paragraph*{目的}
モールドや工具等の幾何学的特性およびマシニング内での運動学的制約を考慮し、加工の際に必要となる位置・距離等の\textbf{解析的な導出}を試みる。
}
\input{./MACN_part_Analytical_Calculation}

\tPart{解析計算に基づく数値解析}{概要}{%
\paragraph*{背景}
一般に、個々の製品や工具等の寸法は異なり、明細ごとに\textbf{固有の寸法}等を持つ。
そのため実際の加工（プログラムの作成）の際には、それら全てに対して\textbf{具体的な数値情報}を指定する必要がある。

　こうした数値情報は明細ごとに膨大にあるが、現時点（\DMname 設置時点）において、こうした手続きは\textbf{明細ごとに手作業}で行われている。

　したがって、こうした手続きをシステム化（自動化）することで、品質はもちろんのこと、作業効率・人的資源・安全(security)・安全(both physical and mental safety)・保守など、いずれの観点からみても能率の低下を大きく抑えることが期待される。
\tcbline*
\paragraph*{目的}
前段階で導出した\textbf{解析的な情報}を用いて、各明細における具体的な\textbf{数値的な情報}に変換するシステムの構築を試みる。
}
\input{./MACN_part_Numerical_Calculation}

\tPart[lot]{諸規程・諸標準の策定：コーディング}{概要}{%
\paragraph*{背景}
あらゆる業務・作業において、何の基準・標準もなく\textbf{管理状態}におくことは不可能である。

　しかし現時点（\DMname 設置時点）において、「\textbf{ソフトウェアエンジニアリング}に関する管理・業務を行う部門」は存在しておらず、専任の担当者さえ存在しない。
そのため、ソフトウェアエンジニアリング視点による\textbf{規程・標準}等も存在しない
%% footnote %%%%%%%%%%%%%%%%%%%%%
\footnote{「ソフトウェア」を「ハードウェア」に置き換えてみると、これがどれほどの異常事態であるかがわかる。}。
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

　したがって、こうした視点による規程・標準を設けることが急務である。
これにより、すべての業務におけるソフトウェア関連作業の管理や、ソフトウェア開発（外注含む）に関する手続き等にかかる（時間的・金額的）コストの増加を大きく抑えられることが期待される。
\tcbline*
\paragraph*{目的}
\DMname についての（特にソフトウェアの観点による）\textbf{規程および標準の策定・制定}を試みる。
}
\input{./MACN_part_Rules}
